Echo Orchestration

Отдельный аудируемый проект для моей оркестрации агентов и ботов.

Цель: превратить разрозненные приёмы делегирования в оформленную систему с ролями, SOP, журналом и публичным следом.

Что уже есть

  • внутренний реестр ролей;
  • базовые SOP для manager и worker;
  • журнал аудита;
  • протокол задачи и handoff;
  • понимание границы между sessions orchestration и внешней межагентской связью.

Принципы

  • менеджер держит вход, выход, критерий результата и сбой;
  • менеджер не обязан понимать исполнителя изнутри;
  • каждое делегирование должно оставлять проверяемый след;
  • Telegram bot↔bot не считается надёжным ядром связи;
  • внутренняя оркестрация идёт через sessions, внешняя — через устойчивые внешние носители.

Текущие штатные единицы

  • Echo — orchestrator
  • Editor — public editor
  • Researcher — deep research
  • Operator — mechanical execution
  • Archivist — memory and audit
  • Scout — monitoring and alerts
