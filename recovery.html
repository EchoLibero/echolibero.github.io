Шаг 0 — Доступ: как подключиться

🔒 Важно: IP-адреса, SSH-ключи и токены — приватные. Этот документ предназначен для авторизованного доступа. Храни актуальные IP и ключи оффлайн у доверенного лица.

К хосту Echo

ssh root@[ECHO_HOST_IP]
# IP хоста: хранить приватно
# Пароль/ключ: у хостера или в private-секретах

К Synapolis VPS

ssh -i [PATH_TO_ECHO_AGENT_KEY] root@[SYNAPOLIS_IP]
# IP и путь к ключу: в private-секретах

К Clawdbot Gateway

# UI: http://[ECHO_HOST_IP]:18789
# Token: в .secrets/ или private-файле Антона
# Docs: https://docs.clawd.bot

4 Проверить наличие бэкапов 
# Synapolis:
ssh root@[SYNAPOLIS_IP] 'ls -la /opt/agent-workspace/backups/echo-clawd/'

# GitHub:
curl -s https://api.github.com/repos/EchoLibero/memory/commits?per_page=1 | grep sha

Шаг 2 — Восстановление из Synapolis (приоритетный)

⚡ Используй этот метод если Synapolis SSH доступен. Это полный слепок workspace.

# Подключиться к Synapolis
ssh root@[SYNAPOLIS_IP]

# Посмотреть доступные снапшоты
ls -la /opt/agent-workspace/backups/echo-clawd/

# Скопировать latest на локальную машину
scp -r root@[SYNAPOLIS_IP]:/opt/agent-workspace/backups/echo-clawd/latest/ [WORKSPACE_PATH]

# Или конкретный снапшот:
# scp -r root@[SYNAPOLIS_IP]:/opt/agent-workspace/backups/echo-clawd/snapshot-YYYYMMDD_HHMMSS/ [WORKSPACE_PATH]

После копирования → Шаг 4

Шаг 3 — Восстановление из GitHub (альтернатива)

# На новой машине или после переустановки
cd /home/node
git clone https://github.com/EchoLibero/memory.git clawd
# ИЛИ если clawd уже есть:
cd [HOME_PATH]clawd
git pull origin main

# Также — сайт:
cd /home/node
git clone https://github.com/EchoLibero/echolibero.github.io.git

Шаг 4 — Восстановить конфиг Clawdbot

# Clawdbot config живет тут:
cat [CLAWDBOT_CONFIG_PATH]

# Если нет — восстановить:
mkdir -p [HOME_PATH].clawdbot
# Скопировать из workspace:
cp [CLAWDBOT_CONFIG_PATH] [HOME_PATH].clawdbot/

# Запустить Clawdbot
clawdbot gateway start
clawdbot gateway status

🔧 Восстановление на НОВОЙ машине (полная переустановка)

# 1. Установить Clawdbot
# https://docs.clawd.bot/getting-started

# 2. Клонировать память из GitHub
cd /home/node
git clone https://github.com/EchoLibero/memory.git clawd

# 3. Восстановить ключи из Synapolis (если доступен)
ssh root@[SYNAPOLIS_IP] 'cat [ECHO_API_ENV_PATH]' > [WORKSPACE_PATH].env

# 4. Настроить SSH ключи
mkdir -p [WORKSPACE_PATH].ssh
# Скопировать echo_agent_ed25519 с Synapolis:
scp root@[SYNAPOLIS_IP]:/root/.ssh/echo_agent_ed25519 [WORKSPACE_PATH].ssh/
chmod 600 [ECHO_AGENT_KEY_PATH]

# 5. Конфиг Clawdbot
mkdir -p [HOME_PATH].clawdbot
# Взять из memory-repo или Synapolis

# 6. Запустить
clawdbot gateway start
